Visão Geral

O Heartthrob é mais um framework feito em CSS 3 e JavaScript, e aqui você pode ver as regras de utilização do Heartthrob, e então, começar a criar páginas incrívies para a web utilizando esse podereso framework.

HTML5

O Heartthrob utiliza elementos do HTML5 e propriedades do CSS que requerem o uso do HTML5 doctype. Inclua-o no início de todos os seus projetos.

Renderização

Para garantir a renderiazação adequada e sensibilidade ao toque, adicione a meta tag viewport dentro de head.

Código usado

Você pode desativar a ampliação do zoom em dispositivos móveis adicionando user-scalable=no na meta tag viewport. Isso desativa o zoom, ou seja, os usuários só são capazes de trabalhar com a barra de rolagem, e o resultado é que seu site ficará parecido com um aplicativo nativo. Entretanto, nós não recomendamos isso em todos os sites, por isso tome cuidado!

Sistema de grid

O Heartthrob inclui um sistema de grid fluido responsivo, mobile first que balança apropriadamente até 12 colunas como dispositivo ou viewport de tamanho aumentado. Ele inclui classes predefinas para opções de layout fácil.

Introdução

Sistema de gride é usado para creação de layout de páginas através de uma série de linhas e colunas que abrigam seu conteúdo. Aqui esta como o sistema de grid do Heartthrob trabalhao:

  • Linhas (row) devem estar localizadas dentro de um conteiner com largura pre definida.

  • Use linhas para criar grupos horizontais de colunas.

  • O conteúdo deve ser localizado dentro das colunas, e apenas colunas podem ser filhas imediatas de linhas.

  • Se mais que 12 colunas são colocadas dentro de uma única linha, cada grupo de colunas extras serão, como uma unidade, colocadas dentro de uma nova linha.

  • Classes de grid aplicadas para dispositivos com largura de tela maiores ou iguais aos tamanhos de ponto de parada, e sobrescreve classes de grid direcionadas em dispositivos menores. Portanto, por exemplo, aplicando qualquer classe .col-md-* para um elementos não irá apenas afetar seu estilo em dispositivos médios mas também em dispositivos grandes se uma classe .col-lg-* não está presente

Formulários

Exemplos básicos

Controles de formulários individuais recebem automaticamente estilos definidos pelo próprio navegador. Todos os elementos textuais input, textarea e select são ajustados para width: 100%, por padrão. Envolva labels e controles em .form-group para melhor espaçamento.

Use a classe .help-block para estilizar o texto de ajuda, como no exemplo abaixo.

Exemplo

Exemplo de texto de ajuda em nível de bloco.


Código usado

<form> <label for="txtMail">Endereço de e-mail</label> <input type="email" id="txtMail" placeholder="Email"> <label for="txtSenha">Senha</label> <input type="password" id="txtSenha" placeholder="Senha"> <label for="fileInput">Entrada de arquivo</label> <input type="file" id="fileInput"> <p class="help-block">Exemplo de texto de ajuda em nível de bloco.</p> <label> <input type="checkbox"> Me selecione </label> <button>Enviar</button> </form>

Formulário em linha

Adicione .form-inline ao seu formulário (não necessáriamente um form) para controles de left-aligned e inline-block. Isso se aplia apenas para formulário dentro de viewports que são pelo menos com 768px de largura.

Larguras personalizadas

Inputs e selects tem width: 100% aplicado por padrão no Heartthrob. Dentro de formulários inline, é redefinido para width:auto, assim vários controles podem residier na mesma linha. Dependendo do seu layout, pode ser necessário adiconar larguras personalizadas.

Sempre adicione labels

Os leitores de tela tersão dificuldade com sues formulários, se você não incluir um label para cada entrada. Para estes formulários in-line, vocêpode ocultar as labels usando a classe .sr-only. Existem outros métodos alternativos de fornecer uma label para tecnologias assistivas, tias como os atributos aria-label, aria-labelledby ou title. Se nenhum desses estiver presente, leitores de tela podem usar o atributo placeholder, se presente, mas note que o usuo do placeholder como um substituto para outros métodos de rotulagem não é aconselhável.

Exemplo

Código usado

<form class="form-inline"> <label for="txtNome2">Nome</label> <input type="text" id="txtNome" placeholder="Cezar Norton"> <label for="txtMail2">Email</label> <input type="email" id="txtMail" placeholder="contato@vtnorton.com"> <button>Enviar convite</button> </form>

Por padrão, navegadores tratarão todos os controles de formulário (elementos input, select e button) dentro de um fieldset desabled como desativado, impedindo interações de mouse e teclado neles. Porém, se seu formulário também inclui elemnetos a ... class="btn btn-*", ester só irá ser dado o estilo de pointer-events:none. Como observado na seção sobre o estado desativado para botões (e especificamente na sub-seção para elementos em âncora), essa propriedade CSS ainda não é padronizada e não é totalmente suportada no Opera 18 pra baixo, ou no Internet Explorer 11, e não irá impedir usuários de teclado de serem capazes de encontrar ou ativar esses links.
